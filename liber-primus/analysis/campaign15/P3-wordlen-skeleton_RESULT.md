# P3 — Word-length Skeleton Attack (Campaign XV)

- **probe_id:** P3-wordlen-skeleton
- **verdict:** null-closed
- **headline_number:** 0 exact and 0 (±1)-tolerance window matches across all 55 unsolved pages vs 11,919 corpus words; real page skeletons score no better than their own length-preserving shuffles (best-fraction z ≈ −1.5…+2.0, none significant).
- **falsifiable_signal:** A corpus passage whose rune-word-length sequence matches a page's skeleton — exactly, or within ±1 at every position — in a window that the permutation (shuffle) null essentially never produces (exact-hit FPR < 1e-3).
- **signal_fired:** false
- **synthetic_validation:** PASS (two independent checks). (1) Embedding a real page-0 skeleton (L=59) into a random corpus: matcher finds it with n_exact=1, best_mismatch=0/59, while 1000 length-preserving shuffles yield exact-hit FPR=0.0000 and never drop below best-fraction 0.593 (mean 0.703). (2) Literal self-match: a genuine 50-word passage from `solved_plaintext.txt` searched against its own corpus → n_exact=1, best_mismatch=0/50, shuffle exact-FPR=0.0000. Matcher demonstrably recovers true plaintext and rejects order-scrambled controls; separation is total (0.000 vs ~0.70/0.30).
- **method:** For each of the 55 unsolved runic pages, split page text on word separators `-` and `.` (strip `/` newlines) and count Gematria runes per word to get an ordered rune-word-length sequence (mean 53.2 words/page, min 17, max 71; total 2,928 words). Convert every candidate plaintext corpus (8 files: `data/keys/*.txt` incl. `solved_plaintext.txt`, `analysis/foundation/*_key.txt` = Hermetica/Sephir Yetzirah/Rosicrucian/Monas/Enoch, and taoteching/thomas/pgp_english_prose) to rune-word lengths using the digraph-compressing `gp.keyword_to_indices` (TH,EO,NG,OE,AE,IA,EA count as one rune). Slide each page skeleton over each corpus with `sliding_window_view`; count windows matching exactly and within ±1 at every position, and record the minimum mismatch fraction. Calibrate a false-positive rate per page with 1000 length-preserving shuffles (permutation null on word ORDER, multiset preserved).
- **script_path:** analysis/campaign15/P3-wordlen-skeleton.py
- **reproduce_cmd:** `python analysis/campaign15/P3-wordlen-skeleton.py`
- **one_paragraph:** The word-boundary metadata is the one channel an OTP cannot encrypt, so if any unsolved LP2 page's plaintext were lifted from a known corpus, that corpus's rune-word-length sequence would have to contain the page's skeleton in order. The matcher was proven sound: it recovers an embedded truth and a real solved-plaintext passage exactly (n_exact=1) while a length-preserving shuffle null produces exact hits at rate 0/1000. Run against all 55 unsolved pages over 11,919 corpus words, it produced **zero** exact matches and **zero** ±1-tolerance matches; the real page skeletons are statistically indistinguishable from their own shuffles (best-fraction ~0.64–0.69 for full pages, z within noise). No page's boundaries index into any tested corpus. This closes the skeleton channel cleanly: LP2's plaintext is not a contiguous passage from any of these corpora (Cicada esoterica, solved LP prose, Tao Te Ching, Gospel of Thomas, PGP prose, Enochian/Hermetic/Kabbalistic key texts), consistent with the standing verdict that the plaintext is an original composition and/or the word boundaries themselves carry no exploitable positional signal. Informative negative — no key to recover.
